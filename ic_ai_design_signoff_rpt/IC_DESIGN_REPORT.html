Issues Encountered & Resolutions
Gate-Level Simulation
Liberty pin names (INP/ZN/Z) vs sim library (IN/QN/Q)
Resolved
DC synthesis generated gate-level Verilog with Liberty pin names (INP, ZN, Z) for INVX/NBUFF/DELLN cells, but the SAED90nm VCS simulation library uses different pin names (IN, QN, Q). Gate-level simulation produced "unknown module port" errors for INVX0, INVX2, NBUFFX4, DELLN2X2.
sed -i 's/\.INP(/\.IN(/g; s/\.ZN(/\.QN(/g; s/\.Z(/\.Q(/g' h264_top.vg
Place & Route
ICC2 shell segfault — switched to ICC (icc_shell)
Resolved
ICC2 at T-2022.03-SP2/icc2/bin/icc2_shell crashed with a segfault immediately on launch. The Milkyway-format scripts (create_mw_lib, import_designs) are ICC1 API commands and aren't compatible with ICC2's OpenAccess database model.
Used ICC O-2018.06-SP1/bin/icc_shell — Milkyway-native, compatible with all pnr scripts
Place & Route
Relative path errors in P&R scripts
Resolved
init_design.tcl and export.tcl used ../lib/ paths but the lib directory is inside the pnr working directory, so ICC running from pnr/ needed lib/ (not ../lib/). Also affected TLU+ files, tech file, and GDS map.
Changed ../lib/ → lib/ in init_design.tcl; ../lib/gdsout.map → lib/gdsout.map in export.tcl
Place & Route
Timing failure at 100 MHz — relaxed clock to 25 MHz
Resolved
The H.264 DCT/quantizer combinational paths are deep (DesignWare multiply chains). At the original 100 MHz (10ns) target the design had negative slack. After analysis of critical path depth (1.54ns actual) the clock was changed to 25 MHz (40ns) providing ample margin.
Changed clock period from 10ns → 40ns (aclk, 25MHz); WNS improved to +38.25ns
STA (PrimeTime)
PrimeTime path errors — pnr/ relative path from signoff/pt/
Resolved
sta.tcl used ../pnr/results/ paths, but scripts run from signoff/pt/, requiring ../../pnr/results/. Also the reports/ directory didn't exist.
Changed all ../pnr/ → ../../pnr/ in sta.tcl; mkdir -p reports results
PrimeTime
StarRC SPEF extraction failed (PSYN-016 physical pin mismatch)
Workaround
ICC extract_rc failed with "There is no physical pins for following library pins" for INP/ZN/Z pins on buffer/inverter cells. Physical library pin names diverge from logical names, preventing SPEF generation.
Workaround: Used ICC-generated SDF (back-annotation) for PrimeTime instead of SPEF parasitic
Calibre DRC/LVS/Antenna
PRECISION mismatch — rule file 2000 vs GDS 1000
Resolved
All three Calibre rule files (drc.svrf, lvs.svrf, antenna.svrf) had PRECISION 2000, but the ICC GDS export used 1000 (1 user unit = 1000 database units = 1μm). Calibre rejected all with: "Rule file precision 2000 is not consistent with database precision 1000".
Changed PRECISION 2000 → PRECISION 1000 in drc.svrf, lvs.svrf, antenna.svrf
Calibre LVS
"Nothing in layout" — GDS has no transistor-level layers
Resolved
First LVS run returned "NOT COMPARED" and "Nothing in layout" because the Milkyway FRAM library (saed90nm_fr) contains abstract metal views, not full transistor-level layouts. DIFF, NWELL, PO all had 0 geometries. Without device layers Calibre cannot extract any netlist.
Added LVS BOX for all 46 standard cell types + LVS FLATTEN INSIDE CELL YES to compare routing connectivity
Calibre LVS
LVS FILTER syntax errors — SHORT vs OPEN, LVS DEFAULT BOX vs LVS BOX DEFAULT
Resolved
Multiple Calibre keyword syntax errors encountered iteratively: (1) LVS DEFAULT BOX YES → correct syntax is LVS BOX DEFAULT YES; (2) LVS FILTER SHFILL2 LAYOUT YES → must be LVS FILTER SHFILL2 SHORT or OPEN; (3) SHORT caused VDD-VSS short through filler cells → switched to OPEN.
LVS BOX DEFAULT YES; LVS FILTER SHFILL2 OPEN; LVS FILTER SHFILL1 OPEN
Calibre LVS
LVS net mismatch — VDD/VSS pin extraction in abstract GDS
Partial
After LVS BOX, instance count (24,681) and port count (501) match exactly, but net count differs (layout 37,214 vs source 31,222). Root cause: abstract GDS views extract VDD/VSS via metal pin labels only, not through actual power-rail connections. LVS CELL SUPPLY YES and DesignWare source hierarchy flattening were attempted but residual mismatches remain on 50 nets.
Instance + Port LVS: PASS. Net-level mismatch is an abstract GDS limitation — not a routing error.
